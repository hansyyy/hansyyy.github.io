DNS

DNS定义

DNS的全称是Domain Name System，域名系统。互联网中识别主机的方式有两种，通过主机名和IP地址。我们人类喜欢用名字方式去记忆，而通信链路中的路由则使用的定长、有层次结构的IP地址。DNS就是一种能提供把主机名转换成IP地址这种服务的由分层DNS服务器实现的分布式数据库。DNS运行在UDP上，使用53端口。

DNS查询

DNS的主要层次：

  1. 根域名服务器：根DNS服务器
  2. 顶级域名服务器：com DNS服务器、org DNS服务器、edu DNS服务器
  3. 权威DNS服务器：amazon.com DNS服务器、pbs.org DNS服务器、mass.edu DNS服务器等等

还有一种是本地域名服务器(local DNS server)。每个 ISP(Internet Service Provider) 比如居民区的 ISP 或者一个机构的 ISP 都有一台本地 DNS 服务器。当主机和 ISP 进行连接时，该 ISP 会提供一台主机的 IP 地址，该主机会具有一台或多台其本地 DNS 服务器的 IP地址。通过访问网络连接，用户能够容易的确定 DNS 服务器的 IP地址。当主机发出 DNS 请求后，该请求被发往本地 DNS 服务器，它起着代理的作用，并将该请求转发到 DNS 服务器层次系统中。

首先，查询请求会先找到本地 DNS 服务器来查询是否包含 IP 地址，如果本地 DNS 无法查询到目标 IP 地址，就会向根域名服务器发起一个 DNS 查询。如果根域名服务器无法告知本地 DNS 服务器下一步需要访问哪个顶级域名服务器，就会使用递归查询；如果根域名服务器能够告知 DNS 服务器下一步需要访问的顶级域名服务器，就会使用迭代查询。在由根域名服务器 -> 顶级域名服务器 -> 权威 DNS 服务器后，由权威服务器告诉本地服务器目标 IP 地址，再由本地 DNS 服务器告诉用户需要访问的 IP 地址。
